# 作業方針

このプロジェクトでは、依頼を受けてもすぐに実装を開始しない。

## 初期フェーズ

以下が確定するまでは、ソースコードの追加・変更を禁止する。

1. 作品の中心となる生成原理
2. システムを構成する要素
3. 要素同士の相互作用
4. 見た目に現れる大域的な振る舞い
5. パラメータと制約
6. 技術的な実現方法の候補
7. 評価方法

初期フェーズでは、調査、提案、比較、設計文書の更新だけを行う。

## 実装開始条件

ユーザーが明示的に「実装を開始してよい」と指示するまで実装しない。

「試しに作る」「最小構成を作る」「確認用に実装する」ことも、
実装開始の指示がない限り行わない。

## 設計の進め方

- 見た目の案だけでなく、見た目を生むルールを記述する
- 単一案へ早期に収束せず、複数の生成原理を比較する
- 一般的な粒子、ノイズ、ブルームの組み合わせで済ませない
- 各案について、予想される現象と破綻条件を説明する
- 未確定事項を勝手に補完せず、設計文書に未決事項として残す
