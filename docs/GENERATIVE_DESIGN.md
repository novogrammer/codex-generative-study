# Generative Design

## 文書の位置づけ

この文書には現在の設計だけを記載する。過去の案や議論の経緯は含めない。

初期実装を開始している。以下の数値は作品として確定した値ではなく、基本構造を確認するための仮値とする。

## 1. 中心となる生成原理

3Dでの基本的な考え方は次の通り。

- 媒介変数と時刻から、円運動系の生成規則によって基準となる3D曲線上の位置を直接生成する
- 根元の断面軸を曲線に沿って回転最小フレームとして運び、断面を配置してチューブ状の表面を作る

円運動から連続曲線を直接生成するため、3次Bスプラインは使用しない。初期実装では二つの円運動の位相を使い、A位相をX座標、B位相をY座標、AとBの合成位相をZ座標へ割り当てた3Dリサージュ型の曲線とする。曲線が有機的に見えることは要件としない。

運動は力学シミュレーションによらず、非力学的な生成規則によって決める。

## 2. システムを構成する要素

- チューブ状の表面を構成する断面列
- 相互作用を持たず、それぞれ異なる位相、半径、速度で動く複数のチューブ
- 深海空間のマリンスノー
- チューブ表面に重ねる金属薄板

チューブの先端と根元は、それぞれの端部断面と同じ位置に半径0の中心頂点を置き、平らな面で閉じる。中心線の全長は変えない。

金属薄板は各チューブに32列×5枚配置し、隣接する列では円周方向へ半枚分ずらす。チューブ表面から0.006m離し、長さ0.22m、幅0.13m、先端側を0.018m持ち上げる。チューブ本体と同じ曲線と回転最小フレームで変形するが、別のInstancedMeshとして描画し、質感を独立して調整する。

## 3. 要素同士の関係

親子関係は持たない。各頂点の位置は媒介変数、断面角、時刻、インスタンス番号から独立に評価する。チューブ同士にも相互作用を持たせない。

## 4. 大域的な振る舞い

異なる平面の回転成分を合成し、単一平面の円や直線を揺らした形に留まらない3D曲線群を作る。個体差を持つ同一生成則が空間内に反復する。

## 5. パラメータと制約

- 座標系では、`position` の1単位を1メートルとする

### 初期実装の仮値

- チューブ数: 32
- 長手分割: 96、断面分割: 12、チューブ半径: 0.1m
- モードA（X座標）: 振幅1.4m、0.50回転、速度0.025回転/秒
- モードB（Y座標）: 振幅0.65m、0.90回転、速度-0.017回転/秒
- 合成位相によるZ振幅: 0.9m
- 位相: 32本のインスタンス番号を0〜2πへ等間隔に割り当て、全体で1周期を構成する。モードAとBは同じ基本位相を使う
- 個体差: 半径と速度に決定論的な変化。変化幅は基準値の±20%
- 配置: 全チューブの始点を原点に固定し、中心から伸びる構成とする
- 個体の向き: X軸±0.45、Y軸±0.7、Z軸±0.35ラジアンの範囲に決定論的に配置

## 6. 技術的な実現方法

- ブラウザで実行する
- ViteとThree.jsを用いる
- ReactなどのUIフレームワークは用いず、プレーンなHTMLを基盤とする
- 実装時は関数中心ではなく、クラス中心の構成を優先する
- `WebGPURenderer` とTSLを使用し、WebGPU非対応環境ではThree.jsのWebGL2バックエンドへフォールバックする
- `InstancedMesh`、`MeshStandardNodeMaterial`、`positionNode`、`normalNode`を使用する
- マリンスノーはカメラ正対する平面の `InstancedMesh` と `MeshLambertNodeMaterial` で描画し、通常のライトを受ける
- CPUによる毎フレームの頂点更新と `SkinnedMesh` は使用しない
- 根元の断面軸はY軸とZ軸から作り、各頂点で根元から16段階の平行移動を独立評価して回転最小フレームを近似する
- 変形後の法線は回転最小フレームの断面放射方向からGPU上で求める
- 変形後の法線にはインスタンスごとの回転を適用し、ビュー空間へ変換して照明計算へ渡す
- 力学シミュレーションは使用しない

## 7. 評価方法

- 32本の曲線が複数の回転成分を持つ3D曲線として読めること
- NaNによる頂点消失や法線反転による激しい明滅がないこと
- WebGPUとWebGL2フォールバックの双方で継続して動作すること
- デスクトップとモバイル相当幅で構図と描画が成立すること
