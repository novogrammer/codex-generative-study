# Generative Design

## 文書の位置づけ

この文書には現在の設計だけを記載する。過去の案や議論の経緯は含めない。

現段階では設計途中であり、実装、擬似コード、確認用プロトタイプは作成しない。

## 1. 中心となる生成原理

3Dでの基本的な考え方は次の通り。

- 媒介変数と時刻から、円運動系の生成規則によって基準となる3D曲線上の位置を直接生成する
- 関節を基準曲線に沿って配置する

円運動から連続曲線を直接生成するため、3次Bスプラインは使用しない。回転軸、半径、周期、時間変化の構成は未決定。曲線が有機的に見えることは要件としない。

運動は力学シミュレーションによらず、非力学的な生成規則によって決める。

## 2. システムを構成する要素

- チューブ状の表面を構成する断面列
- 断面列を変形させる、より疎な変形フレーム列

断面数と変形フレーム数は分けて扱い、1対1にはしない。具体的な数は未決定。

## 3. 要素同士の関係

根元から親子関係を順番に解く方式では、依存する深さに応じて計算段数が増えるという懸念がある。

変形フレームには親子関係を設けず、それぞれの位置と姿勢を独立に決める。断面は近傍の変形フレームを補間して変形する。

それ以外の接続関係と相互作用は未決定。

## 4. 大域的な振る舞い

3Dへ拡張した結果が、単に直線を揺らしているように見えない方法を探る。

それ以外の大域的な振る舞いは未決定。

## 5. パラメータと制約

- 座標系では、`position` の1単位を1メートルとする

それ以外は未決定。

## 6. 技術的な実現方法の候補

- ブラウザで実行する
- ViteとThree.jsを用いる
- ReactなどのUIフレームワークは用いず、プレーンなHTMLを基盤とする
- 実装時は関数中心ではなく、クラス中心の構成を優先する
- GPGPUの利用を検討する
- 親子関係を持たないフラットなボーン配置による `SkinnedMesh` を、変形フレームと断面の補間を実現する候補とする
- 力学シミュレーションは使用しない
